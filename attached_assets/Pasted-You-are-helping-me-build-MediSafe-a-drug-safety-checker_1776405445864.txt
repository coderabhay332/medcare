You are helping me build MediSafe — a drug safety checker web app for Indian patients.

## What I'm Building

MediSafe allows patients to:
1. Create a profile with their medical history, allergies, and current medications
2. Add new medications (via search, photo upload, or manual entry)
3. Get instant safety checks — does the new medicine conflict with existing ones?
4. Get dosage guidance for their condition

## Project Rules (MUST FOLLOW)

### Architecture: Feature-Based Module Structure
Every feature lives under `app/<feature>/` and MUST have exactly these files:
- `<feature>.route.ts` — Express routes only, zero business logic
- `<feature>.controller.ts` — handles req/res, calls services, sends responses
- `<feature>.service.ts` — all business logic and DB interactions
- `<feature>.schema.ts` — Mongoose schema and model
- `<feature>.validation.ts` — express-validator rules
- `<feature>.dto.ts` — TypeScript interfaces and types

### Common Directory (`app/common/`)
- `helper/` — shared utility functions
- `middleware/` — Express middlewares (error handling, rate limiting)
- `services/` — third-party integrations (Passport, DB connection, Claude client)
- `dto/` — shared TypeScript types

### Coding Standards
- ES6 import/export syntax throughout
- Group imports: core Node modules → third-party → path aliases → local
- Always wrap async controller functions with `express-async-handler`
- Always validate request body/params via `express-validator` before controller logic
- Use explicit `type` or `interface` in all DTO files
- Use `Omit<>` or `Pick<>` when extending DTOs for different contexts
- Hash all passwords with bcrypt before saving
- Use path aliases from tsconfig (e.g. `@/user`, `@middlewares`, `@services`) — never use long relative paths like `../../`

### Linting & Formatting
- Project uses eslint + prettier
- Run `prettier . -w` to format
- Run eslint with zero warnings tolerance — fix all warnings, not just errors

### Scripts
- Local dev: `npm run local` or `npm run start`
- Build: `npm run build` (via tsup)

## Core Helpers

### `app/common/helper/saltParser.ts`
```ts
// Input: "Phenylephrine (5mg/5ml) + Chlorpheniramine Maleate (2mg/5ml)"
// Output: ["phenylephrine", "chlorpheniramine"]

import synonyms from '@/data/salt_synonyms.json';

export function parseSalts(composition: string): string[] {
  return composition
    .split('+')
    .map(part => part.replace(/\(.*?\)/g, '').trim().toLowerCase())
    .filter(Boolean)
    .map(salt => synonyms[salt] ?? salt);
}
```

### `app/common/helper/bannedChecker.ts`
```ts
// Subset match: are all salts of a banned entry present in this medicine?
import bannedList from '@/data/banned_fdcs.json';

interface BannedEntry {
  id: number;
  combination: string[];
  gazette_ref: string;
}

export function checkBanned(salts: string[]): BannedEntry[] {
  const saltSet = new Set(salts);
  return (bannedList as BannedEntry[]).filter(entry =>
    entry.combination.every(s => saltSet.has(s))
  );
}
```

### `app/common/services/medicineIndex.ts`
```ts
// Load CSV into Fuse.js on server startup — never hit disk per request
import Fuse from 'fuse.js';
import { parse } from 'csv-parse/sync';
import fs from 'fs';
import { MedicineRecord, MedicineSearchResult } from '@/medicines/medicines.dto';
import { parseSalts } from '@/common/helper/saltParser';

let fuseIndex: Fuse<MedicineSearchResult>;

export function initMedicineIndex(csvPath: string): void {
  const raw = fs.readFileSync(csvPath, 'utf8');
  const records: MedicineRecord[] = parse(raw, { columns: true, skip_empty_lines: true });

  const enriched: MedicineSearchResult[] = records.map(r => ({
    ...r,
    salts: parseSalts(r.composition),
  }));

  fuseIndex = new Fuse(enriched, {
    keys: ['brand_name', 'composition'],
    threshold: 0.3,
    minMatchCharLength: 3,
  });
}

export function searchMedicines(query: string): MedicineSearchResult[] {
  return fuseIndex.search(query, { limit: 10 }).map(r => r.item);
}
```

## Check Service Logic (`app/check/check.service.ts`)
POST /api/check flow:

Get patient from JWT → load currentMedications from DB
For each new medicine:
a. parseSalts(composition) → saltArray
b. LAYER 1 — Banned check (offline, instant):
checkBanned(saltArray) → if match, status = 'banned', include gazette_ref
c. LAYER 2 — Interaction check vs each current medication:
For each currentMed:
- Get salt pairs: newSalt[] × currentSalt[]
- Query OpenFDA first:
GET https://api.fda.gov/drug/label.json?search=drug_interactions:{salt}&limit=1
Parse response text for mentions of the other salt
- If OpenFDA returns nothing → call Claude Haiku:
Prompt: "Do {saltA} and {saltB} interact? Patient conditions: {conditions}.
Reply ONLY as JSON: { interacts: boolean, severity: 'none'|'mild'|'moderate'|'severe', reason: string (max 2 sentences) }"
Parse JSON. Wrap in try/catch — if fails, mark source as 'unavailable'
d. LAYER 3 — Safety advice conflicts:
Compare safety_advice fields of new medicine vs each current medicine
If both have Liver: "UNSAFE" → flag "Combined liver burden"
If both have Kidney: "UNSAFE" → flag "Combined kidney burden"
e. LAYER 4 — Dosage guidance (Claude Haiku):
Prompt: "Patient has {conditions} and is {age} years old.
They are starting {brand} ({composition}).
What is the standard adult dosage? One sentence. Include frequency and meal timing."
Determine overall safe: boolean
→ false if ANY result has status 'banned' or severity 'severe'
Save full check to CheckHistory collection
Return CheckResponseDTO

## Setup Script (`scripts/parseBannedList.ts`)

Run once before starting the server to generate banned_fdcs.json:

```ts
// Usage: npx ts-node scripts/parseBannedList.ts
// Prerequisite: extract PDF text to data/banned_fdcs_raw.txt first (use pdftotext or pdf-parse)

import Anthropic from '@anthropic-ai/sdk';
import fs from 'fs';
import path from 'path';

async function main() {
  const pdfText = fs.readFileSync(path.join(__dirname, '../data/banned_fdcs_raw.txt'), 'utf8');
  const client = new Anthropic();

  const response = await client.messages.create({
    model: 'claude-sonnet-4-5',
    max_tokens: 8000,
    messages: [{
      role: 'user',
      content: `Parse this Indian Ministry of Health banned drugs list into a JSON array.

For each numbered entry extract:
- "id": the serial number (integer)
- "combination": array of drug/salt names (lowercase, no dosages, no salt forms like hydrochloride)
- "gazette_ref": the notification number and date as a single string

Return ONLY a valid JSON array. No markdown, no explanation, no backticks.

Text:
${pdfText}`
    }]
  });

  const json = (response.content[0] as { text: string }).text;
  fs.writeFileSync(path.join(__dirname, '../data/banned_fdcs.json'), json);
  console.log('banned_fdcs.json generated successfully');
}

main();
```

## Vision Extraction (`app/medicines/medicines.service.ts` — scan method)

```ts
async function extractMedicinesFromImage(imageBuffer: Buffer, mimeType: string) {
  const base64 = imageBuffer.toString('base64');

  const response = await claude.messages.create({
    model: SONNET,
    max_tokens: 1000,
    messages: [{
      role: 'user',
      content: [
        {
          type: 'image',
          source: { type: 'base64', media_type: mimeType, data: base64 }
        },
        {
          type: 'text',
          text: `This is either a doctor's prescription or medicine packaging.
Extract all medicine and drug names visible in this image.
Return ONLY a JSON array of strings.
Example: ["Dolo 650", "Ascoril D Plus", "Metformin 500mg"]
If no medicine names are visible, return: []
No explanation. No markdown. Just the JSON array.`
        }
      ]
    }]
  });

  const extracted: string[] = JSON.parse((response.content[0] as { text: string }).text);

  // Fuzzy match each extracted name against medicine DB
  const matched = extracted.map(name => {
    const results = searchMedicines(name);
    return results.length > 0
      ? { ...results[0], confidence: 0.9 }
      : null;
  }).filter(Boolean);

  const unmatched = extracted.filter((_, i) => !matched[i]);

  return { extracted, matched, unmatched };
}
```

## Important Notes

- Never match medicines by brand name — always by normalized salts
- Always check interactions in BOTH directions (saltA→saltB and saltB→saltA)
- Never save new medicines to currentMedications until user confirms after seeing results
- All Claude API calls wrapped in try/catch — degrade gracefully, never crash
- Medicine CSV loaded into memory ONCE at startup via initMedicineIndex()
- ESLint zero warnings — fix all warnings before committing
- Run prettier before every commit
